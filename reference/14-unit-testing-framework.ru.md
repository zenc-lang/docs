# 14. Библиотека для юнит-тестирования


В Zen C есть встроенная библиотека для тестирования, которая позволяет писать юнит-тесты прямо в исходных файлах с помощью ключевого слова `test`.

#### Синтаксис
Блок `test` содержит описательное имя и тело кода для выполнения. Тесты не требуют функции `main` для запуска.

```zc
test "unittest1" {
    "Это юнит-тест";

    let a = 3;
    assert(a > 0, "a должно быть положительным целым числом");

    "unittest1 пройден.";
}
```


#### Запуск тестов

```bash
zc run my_file.zc
```

Вывод показывает каждый тест по имени:
```
  TEST: описательное имя ... OK
  TEST: другой тест ... НЕУДАЧА

1 test(s) failed
```

#### Утверждения

| Функция | Поведение |
|:---|:---|
| `assert(cond, msg)` | Записывает ошибку, переходит к следующему тесту |
| `expect(cond, msg)` | Не фатально — записывает ошибку, но продолжает в том же тесте |

```zc
test "пример" {
    expect(результат != null, "результат не должен быть null");
    expect(код == 200, "статус должен быть 200");
}
```

#### Код возврата

Бинарный файл завершается с количеством неудачных тестов (0 = все успешно).
