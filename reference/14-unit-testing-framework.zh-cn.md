# 14. 单元测试框架


Zen C 包含一个内置测试框架，允许你使用 `test` 关键字直接在源文件中编写单元测试。

#### 语法
`test` 块包含一个描述性名称和要执行的代码主体。测试不需要 `main` 函数即可运行。

```zc
test "unittest1" {
    "这是一个单元测试";

    let a = 3;
    assert(a > 0, "a 应该是一个正整数");

    "unittest1 通过。";
}
```


#### 运行测试

```bash
zc run my_file.zc
```

输出显示每个测试的名称：
```
  TEST: 描述性名称 ... OK
  TEST: 另一个测试 ... 失败

1 test(s) failed
```

#### 断言

| 函数 | 行为 |
|:---|:---|
| `assert(cond, msg)` | 记录失败，继续下一个测试 |
| `expect(cond, msg)` | 非致命 — 记录失败但在同一测试中继续执行 |

```zc
test "示例" {
    expect(结果 != null, "结果不应为 null");
    expect(状态码 == 200, "状态码应为 200");
}
```

#### 退出码

程序以失败测试的数量退出（0 = 全部通过）。
