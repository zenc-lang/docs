# 16. Система диагностики


Zen C предоставляет категоризированную систему диагностики, которая дает гранулярный контроль над предупреждениями компилятора. Это помогает поддерживать высокие стандарты качества кода, уменьшая при этом трение при взаимодействии с внешним кодом на C.

#### Категории диагностики
Предупреждения сгруппированы в логические категории. Каждую категорию можно включить или выключить глобально с помощью флагов компилятора.

| Категория | Описание | По умолчанию |
| :--- | :--- | :--- |
| **`INTEROP`** | Предупреждения, связанные с импортом заголовков C и неопределенными внешними функциями. | **ВЫКЛ** |
| **`PEDANTIC`** | Особо строгие проверки на потенциальные проблемы или качество кода. | **ВЫКЛ** |
| **`UNUSED`** | Предупреждения для определенных, но неиспользуемых переменных, параметров или функций. | **ВКЛ** |
| **`SAFETY`** | Критические предупреждения по безопасности, такие как доступ к нулевому указателю или деление на ноль. | **ВКЛ** |
| **`LOGIC`** | Предупреждения, связанные с логикой, такие как недостижимый код или сравнения констант. | **ВКЛ** |
| **`CONVERSION`** | Предупреждения для неявных или сужающих преобразований типов. | **ВКЛ** |
| **`STYLE`** | Предупреждения о стиле программирования, такие как затенение (shadowing) переменных. | **ВКЛ** |

#### Флаги компилятора
Вы можете управлять диагностикой с помощью флагов `-W` (включить) и `-Wno-` (выключить) , за которыми следует имя категории или конкретный идентификатор диагностики.

##### Флаги категорий
- `-Winterop`: Включает все предупреждения, связанные с интероперабельностью.
- `-Wno-unused`: Отключает предупреждения о неиспользуемых переменных/параметрах.
- `-Wsafety`: Гарантирует, что все проверки безопасности активны.
- `-Wall`: Включает все основные категории диагностики.
- `-Wextra`: Включает еще более строгую диагностику (эквивалент -Wpedantic).

##### Пример использования
```bash
# Компиляция с включенными предупреждениями об интероперабельности C
zc app.zc -Winterop

# Компиляция со всеми предупреждениями, кроме неиспользуемого кода
zc app.zc -Wall -Wno-unused
```

#### Трение при взаимодействии с C
По умолчанию Zen C подавляет предупреждения об "неопределенной функции" для функций, которые, вероятно, находятся в стандартных библиотеках C (категория INTEROP -- ВЫКЛ).
Если вы хотите, чтобы компилятор строго помечал каждую неопределенную функцию (например, чтобы отлавливать опечатки), включите категорию интероперабельности:

```bash
zc main.zc -Winterop
```

При включении компилятор будет давать полезные советы для общих функций C:
```
warning: Undefined function 'abs'
  --> main.zc:5:13
   |
5  |     let x = abs(-5);
   |             ^ here
   |
   = note: If this is a C function, it might need to be whitelisted in 'zenc.json'
```

#### Белый список (Whitelisting)
Если вы часто используете определенную библиотеку C и хотите оставить -Winterop включенным, не отвлекаясь на конкретные функции, вы можете добавить их в `c_function_whitelist` в файле конфигурации `zenc.json`.
